### 4.2 Crear el script entrypoint.sh

El script `entrypoint.sh` es crucial para que N8N funcione correctamente en Heroku. Se encarga de parsear la `DATABASE_URL` y configurar todas las variables necesarias antes de iniciar N8N.

Crea un archivo llamado `entrypoint.sh`:

```bash

  echo "PORT variable not defined, leaving N8N to default port."
else 
  export N8N_PORT="$PORT"; 
  echo "N8N will start on '$PORT'"

# Funci√≥n para parsear DATABASE_URL

  eval $(echo "$1" | sed -e "s#^\(\(.*\)://\)\?\(\([^:@]*\)\(:\(.*\)\)\?@\)\?\([^/?]*\)\(/\(.*\)\)\?#${PREFIX:-URL_}SCHEME='\2' ${PREFIX:-URL_}USER='\4' ${PREFIX:-URL_}PASSWORD='\6' ${PREFIX:-URL_}HOSTPORT='\7' ${PREFIX:-URL_}DATABASE='\9'#")

# Separar host y puerto

export DB_POSTGRESDB_PORT=$N8N_DB_PORT
export DB_POSTGRESDB_DATABASE=$N8N_DB_DATABASE
```

### ¬øQu√© hace este script?

1. **Detecta el puerto**: Heroku asigna din√°micamente un puerto a trav√©s de la variable `$PORT`
2. **Parsea DATABASE_URL**: Extrae usuario, contrase√±a, host, puerto y nombre de base de datos usando una funci√≥n robusta

5. **Inicia N8N**: Ejecuta el servidor con todas las variables configuradas
